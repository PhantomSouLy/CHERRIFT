CHERRIFT v0.3.8 Stability Consolidation Patch

[Why]
- The repo had many patch scripts stacked after each other.
- Several scripts patched UI.bind / UI.open / renderGear / renderWorldPanel repeatedly.
- This caused duplicate event listeners, overwritten button handlers, and unstable UI behavior.

[Fixed]
- Added a final stability layer loaded last before main.js.
- Key buttons are cloned and rebound cleanly after UI.init:
  - Play / Mobile Play
  - Skin prev / next / equip
  - Gear interactions
  - World Select prev / next / play / back
  - Pause / resume / fullscreen / settings
  - Stage clear buttons
  - Back buttons and data-open menu buttons
- Gear is now rendered through one stable renderer.
- Gear drag no longer uses native HTML drag.
- Gear drag uses pointer events with movement threshold.
- Gear item ghost clears on pointerup, pointercancel, blur, and tab hidden.
- World Select always uses the full 10-stage list.
- World Select stage dots are rebuilt from the same full stage list.
- Skin panel is rendered from current CHERRIFT_DATA skin each time.
- Skin splash path/cache logic is preserved.
- Base/Fairy/Beastclaw skill info is no longer copied wrong between skins.

[Kept]
- v0.3.7 preloader remains.
- v0.3.6 map texture fixes remain.
- CHERRIFT branding remains.
- Splash art fixes remain.
