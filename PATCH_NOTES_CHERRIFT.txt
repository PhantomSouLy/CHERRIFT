CHERRIFT v0.3.6a UI / Texture Hotfix

[Fixed]
- Skin splash art now preloads and uses cache.
- Skin splash art no longer starts a fresh image load every time you change skin.
- Stale splash loads no longer overwrite the currently selected skin.
- Ground decoration layer fixed:
  - flowers
  - flower1 / flower2
  - mushroom
  These now draw under Cherry/enemies instead of above them.
- Small rock visual size increased to better match collision.
- Pink slime visibility fixed.
  Important technical fix: enemies with `style` were being drawn as bullets because drawObj checked `style` before `hp`.
- Fairy Cherry right-walk direction fixed by flipping the right walk animation.
- Stage objective HUD moved down/left on mobile so it no longer overlaps HP/XP/timer.

[Changed]
- Skin selector simplified.
- Removed long skin description.
- Removed weapon/info card clutter from the visible layout.
- Removed in-game preview section.
- Skin selector now focuses on:
  - rarity
  - combat type
  - skill name
- Skill row can be tapped to show a small info bubble.

[Added]
- Button press feedback for most UI buttons.
- Small press/glow effect added to:
  - menu buttons
  - carousel arrows
  - world arrows
  - fullscreen buttons
  - upgrade cards
  - inventory / gear buttons
